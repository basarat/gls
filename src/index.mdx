# General Layout System
`gls` is a general purpose maintainable and simple layout componets for react üåπ. 

# CSS concepts
There are a few key CSS concepts (not specific to gls) to maintainable layout in the world of components, *external airspace*, *flexible expansion* and *internal airspace*. Lets tackle them one by one: 

## External Airspace
A fundamental rule (broken quite commonly in the wild sadly) is that a component only renders stuff in its airspace. That is, it does not bleed out resulting in conflicts with its neighbours. 

Consider a simple'ol `div`. We'll give it a background `lightskyblue` to denote its airspace. It can render whatever it wants in its airspace: 

<Playground style={{ backgroundColor: "#eee", margin: '-2em', padding: '2em' }}>
  <div style={{backgroundColor: 'lightskyblue'}}>Hello</div>
</Playground>

Now try and layout two of these divs together, a naive way would be to add margins:  

<Playground style={{ backgroundColor: "#eee", margin: '-2em', padding: '2em' }}>
  <div style={{backgroundColor: "lightskyblue", marginBottom: '24px'}}>Hello</div>
  <div style={{backgroundColor: "lightpink", marginBottom: '24px'}}>World</div>
  <div style={{backgroundColor: "darkorange"}}>Spacing</div>
</Playground>

A fundamental issue with `<div style={{marginBottom: '24px'}}>Hello</div>` component, is that its not useable in isolation. If we render it alone, note that its leaking out *assuming* its going to be used by something that's going to add another sibling after it: 

<Playground style={{ backgroundColor: "#eee", margin: '-2em', padding: '2em' }}>
  <div style={{backgroundColor: "lightskyblue", marginBottom: '24px'}}>Hello</div>
  <i style={{position:'absolute', bottom:'30px'}}>Notice that excess space at the bottom</i>
</Playground>

So instead of putting the responsiblity of *space seperation* on the component, it can be better managed by the container, e.g. Here the container will space each of its children (except the last) using `margin-bottom`: 

```css
parent>* {
  marginBottom: spacing
}
parent>*:last-child {
  marginBottom: 0px,
}
```

Such a container is the `Vertical` component provided by gls. You can see it seperate the children below:

<Playground style={{ backgroundColor: "#eee", margin: '-2em', padding: '2em' }}>
  <Vertical>
    <div style={{backgroundColor: "lightskyblue"}}>Hello</div>
    <div style={{backgroundColor: "lightpink"}}>World</div>
    <div style={{backgroundColor: "darkorange"}}>Spacing</div>
  </Vertical>
</Playground>

And no leaking margins occur with lesser children:

<Playground style={{ backgroundColor: "#eee", margin: '-2em', padding: '2em' }}>
  <Vertical>
    <div style={{backgroundColor: "lightskyblue"}}>Hello</div>
    <div style={{backgroundColor: "lightpink"}}>World</div>
  </Vertical>
</Playground>

Furthermore the `Vertical` component itself doesn't bleed into external airspace, allowing you to compose two `Vertical`s seamlessly: 

<Playground style={{ backgroundColor: "#eee", margin: '-2em', padding: '2em' }}>
  <Vertical>
    <Vertical>
      <div style={{backgroundColor: "lightskyblue"}}>Hello</div>
      <div style={{backgroundColor: "lightpink"}}>World</div>
    </Vertical>
    <Vertical>
      <div style={{backgroundColor: "lightskyblue"}}>Second Hello</div>
      <div style={{backgroundColor: "lightpink"}}>Second World</div>
    </Vertical>
  </Vertical>
</Playground>

Because `margin`s fundamentally result in airspace violation, do not use margins at the root of your components.

> RULE 1: DO NOT USE MARGIN AT THE ROOT OF YOUR COMPONENT

This rule is also what makes it possible for parents to space children with margins.

## Internal Airspace

We've already talked about how child css selectors can be used to manage the space between children using margins. One final thing worth mentioning is that you can use `padding` to seperate the children from the parent. E.g. here is a vertical with a few children: 

<Playground style={{ backgroundColor: "#eee", margin: '-2em', padding: '2em' }}>
  <Vertical>
    <div style={{backgroundColor: "lightskyblue"}}>Hello</div>
    <div style={{backgroundColor: "lightpink"}}>World</div>
  </Vertical>
</Playground>

Right now the children are right next the to parent. Lets add a `black` background on the `Vertical` to see that the children are *flush* with the vertical: 

<Playground style={{ backgroundColor: "#eee", margin: '-2em', padding: '2em' }}>
  <Vertical style={{backgroundColor: 'black'}}>
    <div style={{backgroundColor: "lightskyblue"}}>Hello</div>
    <div style={{backgroundColor: "lightpink"}}>World</div>
  </Vertical>
</Playground>

If we wanted to `Vertical` to space the children from its border, you can use `padding`: 

<Playground style={{ backgroundColor: "#eee", margin: '-2em', padding: '2em' }}>
  <Vertical style={{backgroundColor: 'black', padding: '24px' }}>
    <div style={{backgroundColor: "lightskyblue"}}>Hello</div>
    <div style={{backgroundColor: "lightpink"}}>World</div>
  </Vertical>
</Playground>

> RULE 3: USE `padding` FOR INTERNAL BORDER AIRSPACE

Normally you would have padding at the root e.g. here are two nested verticals, the border airspace is managed by the root, after all, its the root's border: 

<Playground style={{ backgroundColor: "#eee", margin: '-2em', padding: '2em' }}>
  <Vertical style={{backgroundColor: 'black', padding: '24px' }}>
    <Vertical>
      <div style={{backgroundColor: "lightskyblue"}}>Hello</div>
      <div style={{backgroundColor: "lightpink"}}>World</div>
    </Vertical>
    <Vertical>
      <div style={{backgroundColor: "lightskyblue"}}>Second Hello</div>
      <div style={{backgroundColor: "lightpink"}}>Second World</div>
    </Vertical>
  </Vertical>
</Playground>

We call these roots as *islands* (something with a background). A common island would be `body`. Another example of an *island* is a button, which normally has a background. We use padding to seperate its content from its border (internal airspace management):

<Playground style={{ backgroundColor: "#eee", margin: '-2em', padding: '2em' }}>
  <button style={{backgroundColor: 'lightskyblue', fontSize:'24px', padding: '24px'}}>
    Hello world
  </button>
</Playground>

Similarly *islands* exist commonly in root level content containers (close children of `body`). For example the island below has a white background and padding:

<Playground style={{ backgroundColor: "#eee", margin: '-2em', padding: '2em' }}>
  {/** Island */}
  <Vertical style={{backgroundColor: 'white', padding: '24px' }}>
    <Vertical>
      <div style={{backgroundColor: "lightskyblue"}}>Hello</div>
      <div style={{backgroundColor: "lightpink"}}>World</div>
    </Vertical>
    <Horizontal>
      <Flex style={{backgroundColor: "lightskyblue"}}>Second Hello</Flex>
      <Flex style={{backgroundColor: "lightpink"}}>Second World</Flex>
    </Horizontal>
  </Vertical>
</Playground>
