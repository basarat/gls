# Common Types and Props
We have a common set of props that are supported by a number of components. These will be mentioned in the docs for the individual components as well. We present them below with a reference component.

Before we do that, we present some common types referenced by the props.

## BoxUnit
`type BoxUnit = number | string`. 

Examples: 
* 5 (implies 5px)
* '5rem'

## BoxSet

Various versions of providing common shorthand properties

```ts
export type BoxSet =
  /** top,right,left,bottom */
  | BoxUnit
  /** [topBottom,leftRight] */
  | [BoxUnit, BoxUnit]
  /** [top,right,bottom,left] */
  | [BoxUnit, BoxUnit, BoxUnit, BoxUnit]
```


## GLSProps
Objective: A consistent set of customization options. Members:

* `padding?: BoxSet`: The padding. Using it effectively is covered in the [spacing principle][principle-spacing] section.
* `scroll?: 'both' /** default */ | 'vertical' | 'horizontal' | 'disabled'`: specify scrolling behaviours.
* `styles?: (typestyle.types.NestedCSSProperties | falsy)[]`: an array of custom typestyle `style` objects. Any falsy array members will be safely ignored.

From react: 
* `className?: string`: css class name.
* `style?: React.CSSProperties`: any custom react `style` you want to pass in.

For the tag control: 
```ts
tag?: {
  name?: 'div' /** default */ | 'a' | 'button' // etc.
  props?: React.HTMLProps // props passed to the tag
}
```

## SizingProp

The `sizing` prop:

```ts
/** 
 * Specifies sizing interaction
 */
export interface SizingProp {
  /** 
   * Specifies `sizing` interaction
   */
  sizing?:
  | 'content' /** default */
  | 'flex' /** Same as `1` */
  | number;
}
```
It is covered in our [sizing principal][principle-sizing] section.
